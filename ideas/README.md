# 記事アイデアバックログ（`ideas/`）

このフォルダは、対馬モーターサービス blog の **記事アイデアをストックするバックログ** です。
「あとで書きたい」「現場で思いついた」ネタを、思いついた順に Markdown ファイルとして溜めておき、優先度・カテゴリで整理しながら執筆につなげます。

公開記事（`src/content/blog/`）とは別管理です。ここにあるファイルは **ビルド対象外**（サイトには出力されません）。アイデア帳・編集メモとして自由に使えます。

---

## 目的

- 記事ネタを散逸させず一箇所にストックする
- 優先度（`priority`）と状態（`status`）で「次に何を書くか」を判断できるようにする
- 現場メモ・時事フォローを、そのまま執筆の出発点にできる形で残す

---

## ファイル命名規則

```
YYYY-MM-DD-slug.md
```

- `YYYY-MM-DD`：アイデアを **追加した日**（`created` と揃える）
- `slug`：英小文字・数字・ハイフン区切り。記事の内容が分かる短い識別子
- 例：`2026-06-11-12-month-inspection-content.md`

---

## frontmatter スキーマ

各ファイルの先頭に以下の frontmatter を付けます。

```yaml
---
title: "（仮タイトル）"
status: idea          # idea / draft / published
priority: high        # high / medium / low
category: 整備の現場   # 整備の現場 / 越境EC事業 / AI・自動化 / 対馬ライフ
tags: ["タグ1", "タグ2"]
created: 2026-06-11
source: ケンちゃん現場メモ
---

## 概要
（1〜3行の現場メモ・狙い）

## 構成イメージ（任意）
- H2: ...
- H2: ...

## 参考リンク（任意）
-
```

| フィールド | 必須 | 説明 |
|---|---|---|
| `title` | ✅ | 仮タイトル。執筆時に確定 |
| `status` | ✅ | `idea` / `draft` / `published`（後述の遷移を参照） |
| `priority` | ✅ | `high` / `medium` / `low`。次に書く順の判断材料 |
| `category` | ✅ | 公開ブログと同じ 4 カテゴリのいずれか |
| `tags` | 任意 | 記事の主題タグ。SEO・内部リンクの手がかり |
| `created` | ✅ | アイデア追加日（ファイル名の日付と揃える） |
| `source` | 任意 | ネタ元（例：「ケンちゃん現場メモ」「時事フォロー」） |

`category` は公開ブログのカテゴリ定義（`src/consts.ts` / `src/content.config.ts`）と一致させます：
**整備の現場 / 越境EC事業 / AI・自動化 / 対馬ライフ**

---

## ステータス遷移

```
idea  →  draft  →  published
```

- **`idea`**：ネタとして登録しただけの状態。まだ書き始めていない
- **`draft`**：執筆中／下書きができている状態（`src/content/blog/` に下書きを作り始めたら更新）
- **`published`**：公開済み。対応する公開記事の slug を `概要` に追記しておくと追跡しやすい

公開まで完了したアイデアファイルは、`published` にしたうえで残しておくと「何を書いてきたか」の記録になります。

## 既存 `auto_article.yml` パイプラインとの関係

- 現状の自動記事生成パイプライン（`.github/workflows/auto_article.yml`）は **このバックログを自動では読みません**。
- 当面は **手動運用**：このフォルダを見て「次はこれを書こう」と決め、執筆や記事指定に使います。
- 将来的には、`priority: high` かつ `status: idea` のファイルを自動記事生成の **ネタ元として連携する**ことも検討します（未実装）。

## アイデアの追加方法

### ブラウザから直接（GitHub 上で）

1. GitHub のリポジトリで `ideas/` フォルダを開く
2. **Add file → Create new file**
3. ファイル名を `YYYY-MM-DD-slug.md` で入力
4. 上の frontmatter スキーマをコピーして埋める
5. コミット（ブランチを切って PR にしても、直接 main でも運用に合わせて）

### ファイルで追加（ローカル）

1. `ideas/YYYY-MM-DD-slug.md` を作成
2. frontmatter ＋ 概要を記入
3. コミット & push

---

## 運用のコツ

- 思いついたら **粗くても先に登録**（`概要` 1 行でも OK）。精緻化は後から
- `priority: high` が溜まりすぎたら棚卸しして `medium` / `low` に振り直す
- 時事ネタ（自動運転など）は鮮度が命なので、`priority: high` ＋ 早めの着手を意識する
- 関連記事は `概要` 内で slug を相互参照しておくと、内部リンク設計がラクになる
